# SoundTag Function Points

## 1. 产品定位

SoundTag 是一个面向实体物品的声音标签 MVP。用户通过手机识别 NFC 标签，为实体标签绑定一段录音；之后再次靠近同一张 NFC 标签时，应用会直接播放已保存的声音。

## 2. 账号与会话

- 支持手机号验证码登录。
- 开发模式下后端可返回调试验证码，便于本地联调。
- 登录成功后前端保存 JWT 和当前用户信息。
- App 启动时自动恢复本地会话。
- 会话失效或 `/auth/me` 校验失败时自动清理本地登录状态。
- 支持退出登录。

## 3. NFC 标签识别与路由

- 支持通过手机 NFC 读取标签 UID。
- 支持空白新标签，不要求标签已有 NDEF 内容。
- 支持多种 Android NFC 技术兜底读取，包括 `NfcA`、`MifareUltralight`、`NdefFormatable` 和 `Ndef`。
- 识别到新标签时，自动进入录音页面。
- 识别到已有声音的标签时，自动进入播放页面。
- 识别到属于其他账号的标签时，提示当前账号无权访问。
- 提供开发调试入口，可手动输入标签码模拟 NFC 识别。
- 用户侧主要展示“靠近标签 / 录音 / 播放”流程，不直接展示底层标签 UID。

## 4. 录音创建与绑定

- 新 NFC 标签首次识别后进入录音主流程。
- 录音前可为声音命名。
- 录音过程中实时显示录音时长。
- 支持停止录音。
- 停止录音后支持本地试听。
- 试听时显示播放进度。
- 支持重录，重录会清理当前本地预览状态。
- 用户确认后才上传并绑定标签。
- 上传成功后后端创建标签绑定关系和音频记录。
- 录音绑定完成后自动进入播放详情页。

## 5. 音频上传与 OSS 集成

- 后端基于阿里云 STS 签发临时上传凭证。
- 前端基于临时凭证将音频直传 OSS。
- 上传对象按用户、标签和文件名生成对象路径。
- 前端使用 `expo-file-system.uploadAsync` 进行 multipart 上传，规避 React Native FormData 与 OSS 的兼容问题。
- 后端保存音频对象 key、访问地址、MIME 类型、时长、文件大小等元数据。
- 后端对 OSS 私有对象生成签名 GET URL，保证播放端可访问。
- 删除或重录旧音频时，后端会尝试清理旧 OSS 对象。

## 6. 音频播放

- 已绑定声音的标签再次靠近 NFC 时直接进入播放页。
- 播放页自动加载当前标签的最新声音。
- 支持播放和暂停。
- 支持显示声音名称、时长、保存时间和保存状态。
- 支持从播放页进入重新录制流程。
- 统一管理录音模式和播放模式，降低 Android 音频焦点冲突。
- 播放异常会给出用户可见提示，避免未处理 Promise 警告。

## 7. 重新录制与覆盖

- 已有声音的标签支持重新录制。
- 重录完成后会生成一条新的当前音频记录。
- 旧音频记录会被标记为非当前版本。
- 记录页仅展示每张标签当前有效的声音。
- 重录后旧版本不会在用户侧记录页继续显示。

## 8. 我的声音记录

- 提供“我的声音”列表页。
- 列表仅展示当前账号下仍有效的当前声音。
- 每条声音展示声音名称、保存时间和时长。
- 点击声音卡片可进入播放详情。
- 支持下拉刷新。
- 空状态提示用户先靠近 NFC 标签并录制声音。

## 9. 声音重命名

- 录音创建时可填写声音名称。
- “我的声音”列表支持对当前声音重命名。
- 重命名使用页面内弹窗，兼容 Android。
- 重命名后列表即时更新。
- 后端只允许当前用户重命名自己的当前有效声音。
- 声音名称最长 100 字符。

## 10. 声音删除

- “我的声音”列表支持删除当前声音。
- 删除前有确认弹窗。
- 删除后该声音从列表中移除。
- 删除后对应标签会变为空标签，再次靠近时进入录音流程。
- 删除是软删除，音频记录标记为非当前版本。
- 后端会尝试同步删除 OSS 对象。
- 后端只允许当前用户删除自己的当前有效声音。

## 11. 后端 API

### Auth

- `POST /api/v1/auth/request-code`
  - 申请手机号验证码。
  - 开发模式可返回 `debug_code`。

- `POST /api/v1/auth/verify-code`
  - 校验验证码。
  - 创建或更新用户信息。
  - 返回 JWT 和用户资料。

- `GET /api/v1/auth/me`
  - 获取当前登录用户。
  - 依赖 Bearer Token。

### Tags

- `GET /api/v1/tags/{uid}`
  - 查询标签状态。
  - 返回 `new`、`owned` 或 `locked`。
  - 已绑定且属于当前用户时返回最新有效声音。

- `POST /api/v1/tags/uploads/sts`
  - 为指定标签和文件类型签发 OSS 临时上传凭证。
  - 校验标签是否属于当前用户或尚未绑定。

- `POST /api/v1/tags/{uid}/bind`
  - 将已上传音频绑定到标签。
  - 支持首次绑定和覆盖旧声音。
  - 保存声音名称和音频元数据。

### Records

- `GET /api/v1/records`
  - 获取当前用户所有当前有效声音。
  - 不返回历史版本。

- `PATCH /api/v1/records/{record_id}`
  - 重命名当前有效声音。

- `DELETE /api/v1/records/{record_id}`
  - 删除当前有效声音。
  - 删除后标签可重新录音。

## 12. 数据模型

### User

- 用户 ID。
- 手机号。
- 昵称。
- 创建时间。

### PhoneCode

- 手机号。
- 验证码。
- 过期时间。
- 消费时间。
- 创建时间。

### Tag

- 标签 UID。
- 所属用户。
- 创建时间。
- 更新时间。

### AudioRecord

- 记录 ID。
- 标签 ID。
- 所属用户 ID。
- 声音名称。
- OSS 对象 key。
- 文件访问地址。
- MIME 类型。
- 音频时长。
- 文件大小。
- 是否当前有效版本。
- 被替换时间。
- 创建时间。

## 13. 移动端页面

### 登录页

- 手机号输入。
- 获取验证码。
- 输入验证码。
- 昵称输入。
- 登录。
- 展示开发模式调试验证码。

### 靠近标签页

- 扫描 NFC 标签。
- 根据标签状态自动跳转。
- 手动输入标签码调试。
- 展示最近识别状态。

### 录音页

- 输入声音名称。
- 开始录音。
- 停止录音。
- 实时录音时长。
- 本地试听。
- 重录。
- 确认上传并保存。

### 播放页

- 展示当前声音名称。
- 自动播放当前声音。
- 播放 / 暂停。
- 展示时长和保存时间。
- 重新录制入口。
- 刷新当前标签状态。

### 我的声音页

- 展示当前有效声音列表。
- 进入播放详情。
- 重命名。
- 删除。
- 下拉刷新。

## 14. 权限与环境要求

- Android 需要 NFC 权限。
- Android 需要录音权限。
- 真机 NFC 调试需要自定义 Expo Dev Client，不能使用 Expo Go。
- 真机调试时 `EXPO_PUBLIC_API_BASE_URL` 必须指向手机可访问的局域网后端地址。
- 后端需要可访问 PostgreSQL 或兼容的数据库。
- OSS 上传需要配置阿里云 RAM 用户 AccessKey、STS AssumeRole 角色、OSS bucket 和相关权限。

## 15. 用户体验策略

- 用户侧隐藏标签 UID、OSS 对象地址和本地文件路径。
- 以“靠近标签、录制声音、播放声音、管理声音”为主要语言。
- 新标签路径优先进入录音，减少中间判断页面。
- 已有声音路径优先进入播放，降低用户操作成本。
- 删除和危险操作需要确认。
- 网络、播放、上传、NFC 权限等异常均有用户可见提示。

## 16. 当前边界

- 短信验证码仍是 MVP 开发态实现，生产环境需要接入真实短信服务。
- 当前每张标签用户侧只展示一个当前有效声音。
- 历史版本在数据库中保留状态字段，但用户侧记录页不展示历史版本。
- 删除 OSS 对象失败不会阻断业务删除结果。
- iOS NFC 能力依赖 Apple Developer 相关 entitlements 和真机构建配置。
